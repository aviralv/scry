scry web frontend v2

2026-05-22 · engine pivot to @anthropic-ai/claude-agent-sdk · Perplexity-shape GUI · spec approved

Overview

scry becomes a thin server + Perplexity-style GUI on top of the Claude Agent SDK. Six v1 surfaces (search, library, MCPs, registry, onboarding, preferences) share a single engine exposed through two transports: stdout for the CLI (scry "<query>") and fetch streaming for the GUI (scry serve).

What scry keeps
  • The registry (people, projects, routing metadata)
  • The synthesis prompt style ([N] citations, "note disagreements", "prioritize recent")
  • The CLI as a thin headless caller — XDG fix from v0.1.3 preserved
  • scry.config.yaml as the single source of truth
What scry stops doing
  • Per-source query templating (planner.ts) — the SDK's tools handle this
  • MCP connection pooling (mcp-pool.ts) — SDK manages stdio MCPs
  • LLM synthesis HTTP plumbing (synthesizer.ts) — SDK runs the loop
  • Result normalization (normalizer.ts) — Claude reads raw tool_results

Cancellation + resume

Each search creates an AbortController; its signal is forwarded to the SDK. Closing the browser tab triggers req.raw.signal abort → SDK terminates the loop and kills MCP child processes within 1s. For follow-up turns, the client passes the prior sessionId; the server adds resume: sessionId + the session's prior sources to runQuery; [N] numbering continues seamlessly.

Streaming answer with hover-linked [N] superscripts. Source rail builds in arrival order. Library sidebar grouped by Today / Yesterday / Last week / Older. Click a session → GET /api/sessions/:id from SQLite; follow-up resumes via the SDK. Citations stable across follow-up turns.

2 · MCP manager (/mcps)

Table of configured MCP servers with name, command, enabled toggle, last test status. Add/edit/test/delete; every commit goes through atomicWriteConfig. POST /api/mcps/:name/test spawns the server, lists tools, terminates — 5s timeout, process-group SIGTERM/SIGKILL on hang, env-var allowlist (no shell env leak). Env-var refs only; no literal token entry.

3 · Registry editor (/registry)

Two tabs: People (key, name, aliases, identifiers) and Projects (key, name, aliases, routing — slack channels, jira project, confluence cql). PUT /api/registry writes the full registry block atomically; comments outside the block preserved; zod-validated before write.

4 · Onboarding wizard (/onboarding)

  1. Auth check. Probe ANTHROPIC_AUTH_TOKEN + ANTHROPIC_BASE_URL presence. Show shell snippet for .scry.env if missing. Optional "Test token" button does a 1-token API call (rate-limited 1/30s).
  2. MCP setup. Scan PATH for bundled MCPs; toggle each; show env-var names (no values).
  3. Done. Atomic write of scry.config.yaml to ~/.config/scry/; link to search.

Gating is client-side. SPA always loads. If configExists === false, SPA routes to /onboarding; mutating non-onboarding APIs return 409 config-required. No HTTP redirects (would break POSTs and asset fetches).

Key decisions

Engine pivot to @anthropic-ai/claude-agent-sdk
Reading scry's planner showed "deterministic fanout" was always-call-all + per-source query templates. Agent SDK's loop is smarter (skips irrelevant sources), supports follow-up turns, and parallel tool use is native. ~80% of src/core/* deletes.
Single engine, two transports
runQuery() is the sole entry point. CLI streams to stdout; server streams to GUI via fetch streaming. No dual implementations. The XDG fix from v0.1.3 is preserved.
No global process.chdir
v1 of this spec proposed mutating cwd at boot for "stable JSONL paths". Reviewers correctly flagged side effects. SDK Options.cwd (verified at sdk.d.ts:1283) lets us pass cwd explicitly per call. process.cwd() is never mutated by scry.
Fetch streaming, not EventSource
EventSource can't send custom headers, breaking CSRF. Search uses fetch() + body.getReader() to parse text/event-stream-shaped responses with the CSRF header attached.
Citation tracker is session-scoped
Numbering continues across follow-up turns; an [N] from turn 1 is stable for the life of the session. State persists in sessions.sources_json. Unmapped markers degrade to plain text.
SQLite for the sidebar index only
Full transcripts live in the SDK's JSONL files. scry's SQLite stores the index (id, title, summary, sources, timestamps). Writes happen on done only — never per-token. WAL mode at boot.
Cold resume validated
SDK persists JSONL at ~/.claude/projects/<encoded-cwd>/<id>.jsonl. scry passes cwd: ~/.config/scry/ explicitly. Cold restart → click sidebar entry → follow-up still works.
No literal token entry from UI
Env-var references only. UI shows shell snippets for setting tokens in .scry.env. Server never writes secret values to disk based on browser input.

Visual identity

Cool teal accent on near-black background. Inter for sans-serif, JetBrains Mono for code. Dark + light themes via [data-theme="light"] and ~25 CSS variables in web/src/theme/tokens.css. The whole rebrand surface is one file.
